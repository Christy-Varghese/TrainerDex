## Deferred design debt

### 1. Framer Motion motion system
- **What:** Add the motion layer mandated by CLAUDE.md — page fade-in (`y: 16 → 0`),
  card `whileHover={{ scale: 1.02 }}`, list stagger on grids, skeleton→content
  cross-fade.
- **Why:** The app is fully static (0 `framer-motion` usages). With no transitions,
  every navigation reads as a page reload — "fan site," not "official app." This is
  the difference between the 5-minute behavioral loop feeling fluid vs janky.
- **Pros:** Premium feel; satisfies a stated "Do Not" rule; continuity between
  list→detail→back.
- **Cons:** Touches most interactive components; risk of over-animation if not
  constrained to the CLAUDE.md presets.
- **Context:** Do this as a dedicated pass AFTER the next/image + skeleton migration,
  so motion wraps real loading states (skeleton fade-out → content fade-in).
- **Depends on:** Skeleton/loading states (immediate queue item 3).

### 2. DESIGN.md token system (`/design-consultation`)
- **What:** Consolidate scattered CLAUDE.md/AGENTS.md design rules into a real
  DESIGN.md with named tokens (color, type scale, spacing, glass/card patterns,
  motion presets, icon set).
- **Why:** No DESIGN.md exists; tokens live as repeated utility strings. Every new
  component re-derives the system from prose, which is how drift starts.
- **Pros:** Single source of truth; future reviews calibrate against it; faster
  component authoring.
- **Cons:** Upfront effort before it pays off.
- **Context:** Run `/design-consultation`. Capture the dark CSS-var surface system
  already built in `globals.css` and the `typeColor` map as the seed.

### 3. Breadcrumbs on deep pages
- **What:** Add breadcrumb wayfinding to `/pokemon/[dex]`, `/events/[id]`,
  `/news/[slug]`, `/graphics/week/[week]`.
- **Why:** Krug's "trunk test" — on a deep detail page the user has weak sense of
  location. Current pages only have a single "← back" link.
- **Pros:** Orientation; better SEO crumb structure.
- **Cons:** Minor; low urgency.

### 4. Dashboard single visual anchor
- **What:** Homepage hero currently competes with the quick-action button row for
  "first thing seen." Give the hero one dominant anchor (featured event art or a
  live raid spotlight) so the eye lands once.
- **Why:** Pass 1 — "one strong visual anchor" litmus. Right now attention splits.
- **Cons:** Taste call; needs a mockup to decide the anchor.
